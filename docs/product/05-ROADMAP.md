# 路线图与商业

## 一、里程碑

### MVP（约 4 周）—— 打通「告警 → 自排查 → 播报」闭环

| 周次 | 交付 |
|---|---|
| W1 | 告警网关：接 Alertmanager / 夜莺 webhook + 去重分级 |
| W2 | 排障 Agent 接真实工具：Prometheus 查询 + Loki 日志 + k8s 状态（替换 mock）|
| W3 | IM 播报（飞书 / Slack）+ Web 控制台完善（OD 设计稿已就绪）|
| W4 | 端到端联调 + 真实故障场景测试 + 录制 demo |

**验收**：真实 Prometheus 告警 → 60 秒内在 IM 群给出「根因 + 证据链 + 修复方案」。

### V1（+3 个月）—— 自进化壁垒

- **知识自进化闭环**（最高优先级，核心壁垒）
- 复盘报告自动生成
- MCP 工具市场 + 多数据源接入
- 效果评估看板（MTTR / 准确率 / 覆盖率）
- **交付**：开源 Agent 框架 v1.0

### V2（6–12 个月）—— 商业放大

- Runbook 自动执行（带人工审批）
- 多租户 / SSO / 审计
- 私有化部署版本
- 根因知识图谱（跨 incident 复用）

## 二、商业模式：开源 + 企业版（PLG）

- **开源版**（社区，免费）：核心 Agent + 知识库 + IM 播报，自部署。→ 获客、建立信任、收集反馈
- **团队版**（SaaS，按席位）：开源版能力 + 托管 + 多用户协作 + 评估看板
- **企业版**（私有化，年费）：+ SSO / 审计 / 合规 / Runbook 执行 / 专属支持

**为什么开源**：DevOps / SRE 群体高度认可开源（参照 GitLab / HashiCorp / Grafana 路径）；开源获客成本低、信任度高、易形成社区飞轮。

## 三、定价（示意，需市场验证）

| 版本 | 价格 | 适合 |
|---|---|---|
| 社区开源 | 免费 | 个人 / 小团队自部署 |
| 团队 SaaS | ~¥99 / 席位 / 月 | 中小团队，托管省心 |
| 企业私有化 | ~¥10 万+ / 年 | 中大型企业，合规 / 定制 |

> 定价为示意，实际需结合客户访谈和价值量化（MTTR 下降节省的人力成本）校准。

## 四、获客策略

1. **开源获客**：GitHub 开源 Agent 框架，star / issue 驱动，开发者社区传播
2. **技术内容**：写「多 Agent 排障 / MCP / Agentic RAG」深度文章（知乎 / 掘金 / 公众号），建技术品牌
3. **demo 病毒传播**：「告警一响，AI 60 秒查出根因」对比视频，强冲击力、易出圈
4. **社区 → 企业**：开源用户中的企业，自然转化为企业版客户（经典 PLG 路径）

## 五、风险与对策

| 风险 | 等级 | 对策 |
|---|---|---|
| LLM 幻觉导致误诊 | 高 | 引用溯源 + 证据链 + 高危操作人工审批 + 评估闭环持续优化 |
| 接基础设施复杂（每家环境不同）| 高 | MCP 标准化 + 先支持主流（Prometheus / k8s）+ 开源社区贡献连接器 |
| LLM 调用成本 | 中 | 路由分级（简单用小模型/规则，复杂用大模型）+ 缓存 + 本地模型兜底 |
| 大厂下场（云厂商自带 AIOps）| 中 | 中立多云 + 私有部署 + 知识自进化壁垒，打大厂不愿做的细分 |
| 数据安全顾虑 | 中 | 私有化部署 + 脱敏 + 审计，把「安全合规」做成卖点而非阻力 |

## 六、北极星指标

- **MVP**：1 个真实团队部署使用，MTTR 下降可量化
- **V1**：GitHub 1000+ star，10+ 团队试用，首批付费客户
- **V2**：ARR 达可融资量级，形成知识自进化带来的客户切换成本
