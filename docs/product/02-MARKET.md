# 市场与竞品

## 一、行业痛点（为什么现在做）

1. **On-call 反人性**：SRE 半夜被告警叫醒，MTTA（响应时间）+ MTTR（恢复时间）居高不下，疲劳下易二次失误
2. **知识断层**：排障经验散落在 wiki / 工单 / 人脑，核心人员一离职就断档，新人上手慢
3. **AIOps「检测强、诊断弱」**：Datadog 们擅长告警和看板，但「根因到底是什么、怎么修」仍要人对照多个看板交叉推理——这是最耗时的环节
4. **LLM 浪潮的空白**：大模型擅长「读文档 + 多步推理 + 调工具」，但市面运维 AI 多停在「聊天问答」，没真正接基础设施自主排查

## 二、市场规模与趋势

- **全球 AIOps 市场**：量级在百亿美元（2024），年增速 20%+，Gartner 持续列为高增长赛道
- **核心驱动力**：云原生复杂度爆炸（微服务 / k8s / 多云）→ 人力排障跟不上 → 自动化是刚需而非锦上添花
- **LLM 打开的新切口**：传统 AIOps = 规则 + 经典 ML；LLM 让「自然语言推理 + 自主行动」成为可能，催生出**诊断 / 排障 Agent** 这个全新赛道——传统厂商还没补上
- **国内窗口**：信创 + 数据合规推动国产化 AIOps 需求，阿里云百炼等国产大模型基础设施已成熟，正是国产 LLM 运维 Agent 的时间窗

> 注：市场规模为公开报告量级，非精确数字；重点是「赛道大、增速高、且正处于 LLM 带来的范式切换期」。

## 三、竞品分析

### 1. 传统 AIOps / 监控（Datadog / Splunk / Dynatrace / New Relic）
- **强**：数据采集、告警、看板、APM 极成熟
- **弱**：AI 多用于「异常检测 / 告警聚合」，根因诊断与修复仍靠人
- **与谛听**：**互补**——谛听接它们的告警和数据，补上「诊断大脑」，不重复造轮子

### 2. 事件管理 + AI（PagerDuty AIOps / Rootly / Incident.io）
- **强**：事件流程、on-call 调度、AI 辅助生成 incident 总结
- **弱**：AI 偏「流程辅助」（总结/分诊/通知），**不主动接基础设施排查根因**
- **与谛听**：谛听更深一层——自主取证 + 根因推理

### 3. 开源 AIOps Agent（Robusta / Keep / HolmesGPT）
- **强**：开源、接 k8s、有 AI 排障雏形
- **弱**：知识库 / 自进化弱、多绑定单一 LLM、产品化程度低
- **与谛听**：谛听的**知识自进化 + 多 Agent + MCP 工具生态**是明确差异点

### 4. 国产大模型厂商 AIOps（阿里云 / 百度 / 华为云 内部能力）
- **强**：云厂商自有数据 + 渠道
- **弱**：多绑定自家云、不够中立、面向大客户定制
- **与谛听**：谛听可**多云中立 + 私有化部署**，面向不愿被单一云锁定、或要私有部署的中型企业

## 四、差异化矩阵

|---|---|---|---|---|
| 监控 / 告警 | ★★★★★ | 接入 | ★★★ | 接入（不重复造）|
| AI 根因诊断 | ★★ | ★★ | ★★★ | ★★★★ |
| 自主调工具取证 | ★ | ★ | ★★★ | ★★★★ |
| 私有知识库 | ★ | ★★ | ★★ | ★★★★ |

| IM 实时播报 | ★★ | ★★★★ | ★★★ | ★★★★ |
| 国产 / 私有部署 | ✗ | ✗ | 部分 | ★★★★ |

## 五、机会窗口（为什么是我们 / 为什么是现在）

1. **时机**：LLM + Agent 技术（function calling / MCP / 多 Agent 编排）2024–2025 趋于成熟，「自主排查」第一次变得可行；传统 AIOps 还没补上诊断大脑
2. **国产化红利**：国内数据合规 + 信创，国产 LLM 基础设施成熟，国产运维 Agent 有明确政策与市场红利
3. **清晰切入点**：先开源 Agent 框架获客（开发者信任）→ 再企业版（私有部署 / 审计 / SSO）变现，路径经典且可走通
4. **可构建壁垒**：知识自进化形成数据飞轮——客户用得越多知识库越强，迁移成本越高，这是长期护城河
