# 谛听 DiTing · SRE 排障自动驾驶仪

> **听辨故障，自动驾驶排障。**

谛听是一款面向 SRE / DevOps 团队的 **AI 排障自动驾驶仪**：当线上告警触发，多个 Agent 自主排查（查 runbook 知识库 + 调监控 / 日志 / k8s / 工单取证），给出根因与修复方案，并把每次排障结论沉淀回知识库——**越用越强**。

## 一句话定位

**告警 → 主动自排查 → 根因 + 修复 → 知识自进化。**

## 愿景

让每一次故障都被"听见"并快速解决，把 SRE 从夜半被叫醒、翻手册、问老员工的痛苦中解放出来；让团队的运维知识不再只活在个别人脑子里。

## 目标用户

- **主要**：50–500 人科技公司 / 中大型企业的 SRE、DevOps、平台运维团队
- **次要**：DBA、IT 支持、安全运营（SOC）等任何「有告警 + 有知识库 + 有重复排障」的团队

## 四个核心场景

1. **On-call 故障自排查**（核心）：告警 webhook 触发 → Agent 自主取证 → IM 群实时播报 → 根因 + 修复
2. **运维知识库**：多格式（PDF / Word / 图片 OCR）一键入库，带引用溯源的问答
3. **新人 Onboarding**：新人问"这个告警怎么办"，谛听给标准排查路径（基于历史 runbook）
4. **故障复盘 & Runbook 自动化**：自动生成复盘报告；进阶版读 runbook 自动执行修复（带审批）

## 三个核心差异化

1. **主动介入，不是被动问答** —— 告警触发就开干，不用人去问
2. **执行，不止建议** —— 能真调工具取证，进阶版能执行修复（带人工审批）
3. **知识自进化闭环** —— 排障结论自动沉淀，用得越多越强（数据飞轮 = 护城河）

## 技术地基（已验证）

阿里云百炼（`qwen-plus` / `text-embedding-v3` / `gte-rerank-v2` / `qwen-vl-ocr`）+ LangGraph 多 Agent 编排 + MCP 工具协议 + 混合检索 RAG（向量+BM25+rerank）+ Chroma 向量库 + FastAPI。
